# 6.3.5 IP 86h: Extended INQUIRY Data

The Extended INQUIRY Data VPD page (see table 207) provides the application client with a means to obtain information about the logical unit.

## 6.3.5.1 Returned Data - IP 86h: Extended INQUIRY Data

The following data is returned:

- **Byte 0**:
  - Bits 7-5: PERIPHERAL QUALIFIER: 000b
  - Bits 4-0: PERIPHERAL DEVICE TYPE: 01h (Sequential Access Device)
- **Byte 1** - PAGE CODE (86h)
- **Byte 2** - Reserved
- **Byte 3** - PAGE LENGTH (3Ch)
- **Byte 4**:
  - Bits 7-6: ACTIVATE MICROCODE (00b) -- The ACTIVATE MICROCODE field indicates how the device server activates microcode and establishes a unit attention condition when a WRITE BUFFER command (see 5.2.44) with the download microcode mode set to MODE [05h] (WB) -- Download microcode, save, and activate (see 6.7.1.5 on page 427) or MODE [07h] (WB) -- Download microcode with offsets, save, and activate (see 6.7.1.7 on page 427) is processed.

    | Value | Description |
    |-------|------------|
    | 00b | The actions of the device server may or may not be as defined for values 01b or 10b. |
    | 01b-10b | Not supported |
    | 11b | Reserved |

  - Bits 5-3: SPT -- A supported protection type (SPT) field indicates the type of protection the logical unit supports. The SPT field is reserved if the PROTECT bit in Standard Inquiry Data (see 5.2.5.1 on page 85) is set to zero.

    | Value | Protection type supported |
    |-------|--------------------------|
    | 001b | Logical block protection: Logical block protection (see 4.9 on page 43) describes this feature. |
    | others | Reserved |

  - Bit 2: GRD_CHK (0b)
  - Bit 1: APP_CHK (0b)
  - Bit 0: REF_CHK (0b)
- **Byte 5**:
  - Bits 7-6: Reserved
  - Bit 5: UASK_SUP (0b)
  - Bit 4: GROUP_SUP (0b)
  - Bit 3: PRIOR_SUP (0b)
  - Bit 2: HEADSUP (0b)
  - Bit 1: ORDSUP (0b)
  - Bit 0: SIMPSUP (1b) - The device server supports simple queuing.
- **Byte 6**:
  - Bits 7-4: Reserved
  - Bit 3: WU_SUP (0b)
  - Bit 2: CRD_SUP (0b)
  - Bit 1: NV_SUP (0b)
  - Bit 0: V_SUP (0b)
- **Byte 7**:
  - Bits 7-4: Reserved
  - Bit 3: P_I_I_SUP (0b)
  - Bits 2-1: Reserved
  - Bit 0: LUICLR (0b)
- **Byte 8**:
  - Bits 7-4: Reserved
  - Bit 3: R_SUP (0b)
  - Bits 2-1: Reserved
  - Bit 0: CBCS (0b)
- **Byte 9**:
  - Bits 7-4: Reserved
  - Bits 3-0: MULTI I_T NEXUS MICROCODE DOWNLOAD (0b) -- The MULTI I_T NEXUS MICROCODE DOWNLOAD field indicates how the device server handles concurrent attempts to download microcode using the WRITE BUFFER command (see 5.2.38) from multiple I_T nexuses.

    | Value | Description |
    |-------|------------|
    | 0h | The handling of concurrent WRITE BUFFER download microcode operations from multiple I_T nexus is vendor specific. |
    | 1h-3h | Not Supported |
    | 4h-Fh | Reserved. |

- **Bytes 10-11** - EXTENDED SELF-TEST COMPLETION MINUTES (0000h): Not supported.
- **Byte 12**:
  - Bit 7: POA_SUP (Power On Activate Supported)(1b): A WRITE BUFFER command with the MODE field set to 0Dh (see 6.7.1.11) and the PO_ACT bit set to one is supported.
  - Bit 6: HRA_SUP (Hard Reset Activate Supported)(0b): A WRITE BUFFER command with the MODE field set to 0Dh (see 6.7.1.11) and the HR_ACT bit set to one is not supported.
  - Bit 5: VSA_SUP (Vendor Specific Event Activate Supported)(1b): A WRITE BUFFER command with the MODE field set to 0Dh (see 6.7.1.11) and the VSE_ACT bit set to one is supported. The vendor specific event is the detection of medium removal.
  - Bits 4-0: Reserved.
- **Byte 13** - MAXIMUM SUPPORTED SENSE DATA LENGTH (0000h): Not supported.
- **Bytes 14-63** - Reserved
