# 6.4.9 LP 0Dh[01h]: Environmental Reporting (LTO9 and later)

See 5.2.8 -- LOG SELECT - 4Ch and see 5.2.9 -- LOG SENSE - 4Dh for directions on how to use this page. This page is for counters specific to tape drives.

## 6.4.9.1 Parameter Reset Behavior

## 6.4.9.2 Parameter Definitions

> Log pages may have different reset behaviors. Care should be taken to ensure counters are processed appropriately, including consideration of their reset behaviors. While reset behaviors may be different between products, an attempt is made to maintain compatibility between generations of a given product. See 6.4 -- Log Parameters (LP)

> The parameters in log pages should be dynamically parsed as some parameters may not be present and new parameters may be inserted. The relative location of parameters have changed and are anticipated to continue to change.

| Code | Counter: Description | Length |
|------|----------------------|--------|
| 0000h | **Deck Temperature {LP0Dh[01h]:0000h}:** Information related to the reporting of temperature. See 6.4.9.2.1 -- Temperature Report parameter data for the layout of this parameter. | 8 |
| 0001h to 00FFh | **Additional Component Temperatures {LP0Dh[01h]:00001h to 00FFh}:** Some products may report additional component temperatures at any parameter in this range. See 6.4.9.2.1 -- Temperature Report parameter data for the layout of these parameters. | 8 |
| 0100h | **Deck Relative Humidity {LP0Dh[01h]:0100h}:** Information related to the reporting of relative humidity. See 6.4.9.2.1 -- Temperature Report parameter data for the layout of this parameter. | 8 |

### 6.4.9.2.1 Temperature Report parameter data

The Temperature Report parameter data has the layout shown in table 240.

```
Table 240 -- Temperature Report parameter data layout

           Bit
Byte  7 msb    6       5       4       3       2       1     0 lsb
 0                          Reserved                          MTV
 1                          TEMPERATURE
 2                    LIFETIME MAXIMUM TEMPERATURE
 3                    LIFETIME MINIMUM TEMPERATURE
 4                 MAXIMUM TEMPERATURE SINCE POWER ON
 5                 MINIMUM TEMPERATURE SINCE POWER ON
 6                    MAXIMUM MOUNTED TEMPERATURE
 7                    MINIMUM MOUNTED TEMPERATURE
```

The temperature values reported in the Temperature Report parameter data indicate a temperature in degrees Celsius. Negative values are indicated by two's complement notation. A value of -128 (i.e., 80h) specifies that temperature is not valid.

- **Byte 0:**
  - Bits 7-1: Reserved
  - Bit 0: MTV (mounted temperature valid): Indicates if the MAXIMUM MOUNTED TEMPERATURE field and the MINIMUM MOUNTED TEMPERATURE field are valid.
    - 0b -- Fields are invalid
    - 1b -- Fields are valid
- **Byte 1:** TEMPERATURE: The temperature in degrees Celsius most recently read by the temperature sensor.
  - 80h -- There is no valid temperature to report.
  - other -- The temperature in two's complement notation (i.e., signed).
- **Byte 2:** LIFETIME MAXIMUM TEMPERATURE: The maximum temperature in degrees Celsius read by the temperature sensor over the life of the drive.
  - 80h -- There is no valid temperature to report.
  - other -- The temperature in two's complement notation (i.e., signed).
- **Byte 3:** LIFETIME MINIMUM TEMPERATURE: The minimum temperature in degrees Celsius read by the temperature sensor over the life of the drive.
  - 80h -- There is no valid temperature to report.
  - other -- The temperature in two's complement notation (i.e., signed).
- **Byte 4:** MAXIMUM TEMPERATURE SINCE POWER ON: The maximum temperature in degrees Celsius read by the temperature sensor since power on (i.e., the last power cycle).
  - 80h -- There is no valid temperature to report.
  - other -- The temperature in two's complement notation (i.e., signed).
- **Byte 5:** MINIMUM TEMPERATURE SINCE POWER ON: The minimum temperature in degrees Celsius read by the temperature sensor since power on (i.e., the last power cycle).
  - 80h -- There is no valid temperature to report.
  - other -- The temperature in two's complement notation (i.e., signed).
- **Byte 6:** MAXIMUM MOUNTED TEMPERATURE: The meaning of this field depends on the value of the MTV bit.
  - MTV=0b -- This field is ignored.
  - MTV=1b -- The maximum temperature detected from the most recent time that the volume was mounted until:
    - A) the current time, if the volume has not been demounted; or
    - B) the time at which the volume was demounted.
- **Byte 7:** MINIMUM MOUNTED TEMPERATURE: The meaning of this field depends on the value of the MTV bit.
  - MTV=0b -- This field is ignored.
  - MTV=1b -- The minimum temperature detected from the most recent time that the volume was mounted until:
    - A) the current time, if the volume has not been demounted; or
    - B) the time at which the volume was demounted.

### 6.4.9.2.2 Relative Humidity Report parameter data

The Relative Humidity Report parameter data has the layout shown in table 241.

```
Table 241 -- Relative Humidity Report parameter data layout

           Bit
Byte  7 msb    6       5       4       3       2       1     0 lsb
 0                          Reserved                          MRHV
 1                        RELATIVE HUMIDITY
 2                 LIFETIME MAXIMUM RELATIVE HUMIDITY
 3                 LIFETIME MINIMUM RELATIVE HUMIDITY
 4              MAXIMUM RELATIVE HUMIDITY SINCE POWER ON
 5              MINIMUM RELATIVE HUMIDITY SINCE POWER ON
 6                 MAXIMUM MOUNTED RELATIVE HUMIDITY
 7                 MINIMUM MOUNTED RELATIVE HUMIDITY
```

- **Byte 0:**
  - Bits 7-1: Reserved
  - Bit 0: MRHV (mounted relative humidity valid): Indicates if the MAXIMUM MOUNTED RELATIVE HUMIDITY field and the MINIMUM MOUNTED RELATIVE HUMIDITY field are valid.
    - 0b -- Fields are invalid
    - 1b -- Fields are valid
- **Byte 1:** RELATIVE HUMIDITY: The relative humidity most recently read from the humidity sensor.
  - 0-100 -- Relative humidity
  - 255 -- No valid relative humidity to report
  - others -- Reserved
- **Byte 2:** LIFETIME MAXIMUM RELATIVE HUMIDITY: The maximum relative humidity read by the humidity sensor over the life of the drive.
  - 0-100 -- Relative humidity
  - 255 -- No valid relative humidity to report
  - others -- Reserved
- **Byte 3:** LIFETIME MINIMUM RELATIVE HUMIDITY: The minimum relative humidity read by the humidity sensor over the life of the drive.
  - 0-100 -- Relative humidity
  - 255 -- No valid relative humidity to report
  - others -- Reserved
- **Byte 4:** MAXIMUM RELATIVE HUMIDITY SINCE POWER ON: The maximum relative humidity read by the humidity sensor since power on (i.e., the last power cycle).
  - 0-100 -- Relative humidity
  - 255 -- No valid relative humidity to report
  - others -- Reserved
- **Byte 5:** MINIMUM RELATIVE HUMIDITY SINCE POWER ON: The minimum relative humidity read by the humidity sensor since power on (i.e., the last power cycle).
  - 0-100 -- Relative humidity
  - 255 -- No valid relative humidity to report
  - others -- Reserved
- **Byte 6:** MAXIMUM MOUNTED RELATIVE HUMIDITY: The meaning of this field depends on the value of the MRHV bit.
  - MRHV=0b -- This field is ignored.
  - MRHV=1b -- The maximum relative humidity detected from the most recent time that the volume was mounted until:
    - A) the current time, if the volume has not been demounted; or
    - B) the time at which the volume was demounted.
- **Byte 7:** MINIMUM MOUNTED RELATIVE HUMIDITY: The meaning of this field depends on the value of the MRHV bit.
  - MRHV=0b -- This field is ignored.
  - MRHV=1b -- The minimum relative humidity detected from the most recent time that the volume was mounted until:
    - A) the current time, if the volume has not been demounted; or
    - B) the time at which the volume was demounted.
