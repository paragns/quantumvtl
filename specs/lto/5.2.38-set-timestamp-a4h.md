# 5.2.38 SET TIMESTAMP - A4h[0Fh]

The SET TIMESTAMP command is defined in SPC-4 (see the latest version at
http://www.t10.org/drafts.htm#SCSI3_CMNDSETS). This clause decribes the specific implementation.
The SET TIMESTAMP command (see table 141) requests the drive to initialize the timestamp (see 4.17), if the
SCSIP bit is set to one or the TCMOS bit is set to one in MP 0Ah[01h]: Control Extension (see 6.6.8 on page 374).
If the SCSIP bit is set to zero, the SET TIMESTAMP command is terminated with CHECK CONDITION status,
with the sense key set to ILLEGAL REQUEST, and the additional sense code set to INVALID FIELD IN CDB.

```
                                     Table 141 — SET TIMESTAMP CDB

                                                             Bit
    Byte
                   7          6           5           4                3            2             1       0
     0                                            OPERATION CODE (A4h)

     1                    Reserved                                         SERVICE ACTION (0Fh)

     2
                                                           Reserved
     5
     6           (MSB)
                                                  PARAMETER LIST LENGTH
     9                                                                                                  (LSB)
     10                                                    Reserved
     11                                          Control Byte (see 5.1.2.3)
```

The following paramters apply:

- **PARAMETER LIST LENGTH**: The length in bytes of the SET TIMESTAMP parameters that is transferred from the application client to the device server. A PARAMETER LIST LENGTH of zero indicates that no data is transferred, and that no change is made to the timestamp. Allowed values are 000Ch and 0000h.

## 5.2.38.1 SET TIMESTAMP Parameter List

The layout for the parameter data for the SET TIMESTAMP command is shown in table 142.

```
                          Table 142 — SET TIMESTAMP parameter list layout

                                                              Bit
    Byte
                   7          6           5            4               3            2             1       0
      0
                                                           Reserved
      3
      4          (MSB)
                                                           TIMESTAMP
      9                                                                                                 (LSB)
     10
                                                           Reserved
     11
```

The definition of the parameters follows:

| Byte | Description |
|------|-------------|
| 0-3 | Reserved |
| 4-9 | TIMESTAMP: The initial value of the timestamp in the layout defined in Device Clocks (see 4.17 on page 61). The timestamp should be the number of milliseconds that have elapsed since midnight, 1 January 1970 UT. If the high order byte in the TIMESTAMP field is greater than F0h, the command is terminated with CHECK CONDITION status, with the sense key set to ILLEGAL REQUEST, and the additional sense code set to INVALID FIELD IN PARAMETER LIST. |
| 10-11 | Reserved |

On successful completion of a SET TIMESTAMP command the drive generates a UNIT ATTENTION condition
for the initiator port associated with every I_T nexus except the I_T nexus on which the SET TIMESTAMP
command was received, with the additional sense code set to TIMESTAMP CHANGED.
