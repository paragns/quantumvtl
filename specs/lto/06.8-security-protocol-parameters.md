# 6.8 Security Protocol Parameters (SPP)

Security Protocol parameters are used by the SECURITY PROTOCOL IN (SPIN) - A2h (see 5.2.34 on page 169)
command and by the SECURITY PROTOCOL OUT (SPOUT) - B5h (see 5.2.35 on page 170) command.
The following terms are used in this clause:

| Term | Description |
|------|-------------|
| SPIN | Security Protocol In |
| SPOUT | Security Protocol Out |

## 6.8.1 SPIN Pages (00h - Security Protocol Information)

See 5.2.34—SECURITY PROTOCOL IN (SPIN) - A2h for a description of how to request this page. The Security
Protocol Specific field of the SPIN CDB is defined by Table 384

| Code | Reference |
|------|-----------|
| 0000h | SPIN (00h[0000h]) - Supported Security Protocols List (see 6.8.1.1 on page 444) |

| 0002h | SPIN (00h[0002h]) - Security Compliance Information (see 6.8.1.3 on page 446) |
| 0003h - FFFFh | Reserved |

- **COMPLIANCE DESCRIPTOR MODULE NAME**: Null terminated, null padded data that indicates the name or identifier of the cryptographic module, as reported by NIST or CSEC.

The I_T NEXUS SCOPE field shall contain the value from the logical block encryption scope saved for the I_T_L

established in the device entity. In this case, the KEY DESCRIPTOR field shall be set to the nonce value established
