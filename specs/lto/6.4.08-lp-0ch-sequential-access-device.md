# 6.4.8 LP 0Ch: Sequential-Access Device

See 5.2.8 -- LOG SELECT - 4Ch and see 5.2.9 -- LOG SENSE - 4Dh for directions on how to use this page. This page is for counters specific to tape drives.

## 6.4.8.1 Parameter Definitions (0Ch)

> Log pages may have different reset behaviors. Care should be taken to ensure counters are processed appropriately, including consideration of their reset behaviors. While reset behaviors may be different between products, an attempt is made to maintain compatibility between generations of a given product. See 6.4 -- Log Parameters (LP)

> The parameters in log pages should be dynamically parsed as some parameters may not be present and new parameters may be inserted. The relative location of parameters have changed and are anticipated to continue to change.

| Code | Counter: Description | Length |
|------|----------------------|--------|
| 0000h | **Total Channel Write Bytes {LP0Ch:0000h}:** The total number of bytes of data written from the host on this mount. | 8 |
| 0001h | **Total Device Write Bytes {LP0Ch:0001h}:** The total number of bytes of data written to tape on this mount, not counting ECC and formatting overhead. This is the number of data bytes after compression. | 8 |
| 0002h | **Total Device Read Bytes {LP0Ch:0002h}:** The total number of bytes of data read from tape on this mount, not counting ECC and formatting overhead. This is the number of compressed data bytes read from media before decompression. | 8 |
| 0003h | **Total Channel Read Bytes {LP0Ch:0003h}:** The total number of bytes of data read to the host on this mount. | 8 |
| 0004h | **Approximate native capacity from BOP to EOD {LP0Ch:0004h}:** This is in megabytes (i.e., 10^6). This is not sensitive to the current position of the medium. The approximate native capacity between EOD and EW is the difference of parameter 0005h and this parameter. Conditions may occur that reduce the amount of data that is written before reaching EW. EOD may be beyond LEOP. A value of all bits set to one indicates that this information is invalid (e.g., no volume is mounted, EOD information needs to be rebuilt). | 4 |
| 0005h | **Approximate native capacity between BOP and EW of the current partition {LP0Ch:0005h}:** This is in megabytes (i.e., 10^6). If no volume is mounted or this value is unknown the device server shall set all bits in this parameter to one. | 4 |
| 0006h | **Minimum native capacity between EW and LEOP of the current partition {LP0Ch:0006h}:** This is in megabytes (i.e., 10^6). If no volume is mounted the device server shall set all bits in this parameter to one. | 4 |
| 0007h | **Approximate native capacity from BOP to the current position of the medium {LP0Ch:0007h}:** This is in megabytes (i.e., 10^6). If no volume is mounted the device server shall set all bits in this parameter to one. | 4 |
| 0008h | **Maximum native capacity that is currently allowed to be in the device object buffer {LP0Ch:0008h}:** This is in megabytes (i.e., 10^6). This value may change depending on the current position of the medium (e.g., available native capacity may decrease as the current position of the medium approaches LEOP). | 4 |
| 0100h | **Cleaning Requested {LP0Ch:0100h}:** A non-zero value indicates a cleaning action is requested by the drive. | 8 |
| 8000h | **Total Megabytes Processed Since Cleaning {LP0Ch:8000h}:** Number of megabytes (10^6) processed to tape since last cleaning (written after compression/read before decompression) | 4 |
| 8001h | **Lifetime load cycles {LP0Ch:8001h}:** This is the number of times the drive has been loaded in its lifetime. | 4 |
| 8002h | **Lifetime cleaning cycles {LP0Ch:8002h}:** This is the number of times over its lifetime the drive has been cleaned using a cleaner cartridge. | 4 |
| 8003h | **Lifetime Power-on time {LP0Ch:8003h}:** This is the number of seconds the drive has been powered on over its lifetime. | 4 |
