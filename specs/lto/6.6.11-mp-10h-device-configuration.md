# 6.6.11 MP 10h: Device Configuration

See 5.2.10 for how to set these parameters and 5.2.11 for how to read these parameters.

Mode Page Behaviors (see 4.7 on page 40) describes this device's non-standard behaviors related to mode parameters.

The Device Configuration mode page is used to specify the configuration of items specific to tape drives.

The mode page policy for this page is described in 4.7.1.

```
Table 339 — MP 10h Device Configuration mode page

  5                            READ OBJECT BUFFER EMPTY RATIO

                                     WRITE DELAY TIME
  7                                                                            (LSB)

  9                                       Obsolete
 10              EOD DEFINED            EEG       SEW      SWP        BAML      BAM

                              OBJECT BUFFER SIZE AT EARLY WARNING

 15             WTRE           OIR         REWIND ON RESET          ASOCWP   PERSWP  PRMWP
```

- **Byte 0**:
  - Bit 7: **PS** (Parameter Saveable): 1b. Ignored in MODE SELECT. See 4.7.2.2.
  - Bit 6: **SPF**: 0b
  - Bits 5-0: **PAGE CODE**: 10h
- **Byte 1**: PAGE LENGTH: 0Eh
- **Byte 2**:
  - Bit 7: Reserved
  - Bit 6: **CAP** (Change Active Partition): 0b (Obsolete - Refer to the LOCATE command to change partition) (non-changeable)
  - Bit 5: **CAF** (Change Active Format): 0b (non-changeable)
  - Bits 4-0: **ACTIVE FORMAT**: 00000b (non-changeable)
- **Byte 3**: ACTIVE PARTITION (non-changeable)
- **Byte 4**: WRITE BUFFER FULL RATIO: 00h (value not specified) (non-changeable)
- **Byte 5**: READ BUFFER EMPTY RATIO: 00h (value not specified) (non-changeable)
- **Bytes 6-7**: WRITE DELAY TIME: 012Ch (30 sec) (changeable)
  - The maximum time, in 100 ms increments, that the device server waits before any buffered data that is to be written, is forced to the medium after the last buffered WRITE command that did not cause the object buffer to exceed the write object buffer full ratio.

  > **WARNING**: Changing the WRITE DELAY TIME may result in adverse performance.

- **Byte 8**:
  - Bit 7: **OBR** (Object Buffer Recovery): 0b (non-changeable)
  - Bit 6: **LOIS** (Logical Object Identifier Supported): 1b (non-changeable)
  - Bit 5: Obsolete
  - Bit 4: **AVC** (Automatic Velocity Control): 0b (non-changeable)
    - The speed chosen is defined by the SPEED field in the mode parameter header.
  - Bits 3-2: **SOCF** (Stop on Consecutive Filemarks): 00b (non-changeable)
    - Read ahead to fill buffer, without regard for filemarks
  - Bit 1: **RBO** (Recover Buffer Order): 0b (non-changeable)
  - Bit 0: **REW** (Report Early Warning on Read commands): 0b (non-changeable)
- **Byte 9**: GAP SIZE: 00h (non-changeable)
- **Byte 10**:
  - Bits 7-5: **EOD DEFINED** (End Of Data Defined): 000b (non-changeable)
  - Bit 4: **EEG** (Enable EOD Generation): 1b (non-changeable)
  - Bit 3: **SEW** (Synchronize at Early-Warning): 0b (non-changeable)
    - The device may retain unwritten buffered logical objects in the object buffer when positioned between early-warning and EOP.
  - Bit 2: **SWP** (Soft Write Protect): 0b (non-changeable)
  - Bit 1: **BAML** (Block Address Mode Lock): 0b (non-changeable)
  - Bit 0: **BAM** (Block Address Mode): 0b (non-changeable)
- **Bytes 11-13**: OBJECT BUFFER SIZE AT EARLY WARNING: 000000h (non-changeable)
- **Byte 14**: SELECT DATA COMPRESSION ALGORITHM: 01h (changeable)

  | Value | Description |
  |-------|-------------|
  | 00h | No compression used |
  | 01 | Use default compression algorithm |

  > NOTE 50 - The only advantage to disabling data compression is predictable full tape capacity (see 2.1.39)

  > NOTE 51 - On a MODE SENSE, the value of byte 14 is consistent with what is found in Mode Page 0F, Byte 2, Bit 7. If this byte alone is updated on a MODE SELECT, and Mode Page 0F is not sent, then Mode Page 0F, Byte 2, Bit 7 is updated according to this field. If both Page 10 and Page 0F are sent, then if either setting is enabled, then compression is enabled and both fields updated to reflect enablement.

- **Byte 15**:
  - Bits 7-6: **WTRE** (WORM Tamper Read Enable): 10b (changeable)

    | Value | Description |
    |-------|-------------|
    | 00b | The device treats a value of 00b the same as if the value were 10b. |
    | 01b | Detection of compromised integrity on a WORM medium shall not affect processing of a task. |
    | 10b | If the drive detects compromised integrity on a WORM medium it returns CHECK CONDITION status with the sense key set to MEDIUM ERROR and the additional sense code set to WORM MEDIUM - INTEGRITY CHECK. (3/300Dh). The position of the medium may have changed. |
    | 11b | Reserved |

    > NOTE 52 - An application client should not set the WTRE bit to 01b except for the recovery of data from a WORM medium where the integrity of the stored data has been compromised.

  - Bit 5: **OIR** (Only If Reserved) 0b (changeable-saveable)
    - This field dictates the behavior of commands other than PERSISTENT RESERVE IN, PERSISTENT RESERVE OUT, RESERVE, and RELEASE.

    | Value | Description |
    |-------|-------------|
    | 0b | Commands processed normally when no reservation exists per RVC column of table 30, Supported Common SCSI Commands, on page 76. |
    | 1b | Commands listed with a 'Y' in the RVC column of table 30 are rejected with ILLEGAL REQUEST, NOT RESERVED if received and no reservation is present in the drive. If a reservation is present in the drive, the commands are processed only if a reservation exists that allows access via the I_T nexus from which the command is received. |

  - Bits 4-3: **REWIND ON RESET**: 10b (non-changeable)
    - The position on medium is not changed due to a logical unit reset.
  - Bit 2: **ASOCWP** (Associated Write Protect): 0b (non-changeable)
  - Bit 1: **PERSWP** (Persistent Write Protect) 0b (non-changeable)
  - Bit 0: **PRMWP** (Permanent Write Protect): 0b (non-changeable)
