# 6.4.20 LP 31h: Tape Capacity

See 5.2.8--LOG SELECT - 4Ch and see 5.2.9--LOG SENSE - 4Dh for directions on how to use this page.

> **WARNING:** This vendor specific log page is obsolete and is being deprecated in favor of LP 17h: Volume Statistics (see 6.4.14 on page 301) which is defined in SSC-4. It is still returned with correct values but its not being enhanced to show additional partitions that are supported from LTO6 forward. This log page may be removed in the future. It is highly recommended that use of this log page be removed and replaced with using the Volume Statistics log page.

## 6.4.20.1 Parameter Reset Behavior (31h)

> **WARNING:** Log pages may have different reset behaviors. Care should be taken to ensure counters are processed appropriately, including consideration of their reset behaviors. While reset behaviors may be different between products, an attempt is made to maintain compatibility between generations of a given product. See 6.4--Log Parameters (LP)

This page reflects the currently loaded volume. Values change after a successful format of the volume.

## 6.4.20.2 Parameter Definitions (31h)

> **WARNING:** The parameters in log pages should be dynamically parsed as some parameters may not be present and new parameters may be inserted. The relative location of parameters have changed and are anticipated to continue to change.

| Code | Counter: Description | Size |
|------|----------------------|------|
| 0001h | **Partition 0 Remaining Capacity {LP31h:0001h}**: A representation of the location on medium of EOD expressed in Mebibytes (2^20) Native Capacity between EOD and EOP 0. This is intended to be used as a gauge to display the ratio of partition 0 used. There is no guarantee that the amount of data expressed in this parameter is available for writing. | 4 |
| 0002h | **Partition 1 Remaining Capacity {LP31h:0002h}**: A representation of the location on medium of EOD expressed in Mebibytes (2^20) Native Capacity between EOD and EOP 1. This is intended to be used as a gauge to display the ratio of partition 1 used. There is no guarantee that the amount of data expressed in this parameter is available for writing. | 4 |
| 0003h | **Partition 0 Maximum Capacity {LP31h:0003h}**: Value representing the Native Capacity of partition 0 in Mebibytes (2^20). Note that the value changes depending on how the volume is partitioned. See 4.4--Volume partitioning for size limitations. | 4 |
| 0004h | **Partition 1 Maximum Capacity {LP31h:0004h}**: Value representing the Native Capacity of partition 1 in Mebibytes (2^20). Note that the value changes depending on how the volume is partitioned. The maximum size of partition 1 is determined by the minimum size of partition 0 and the size of formatting overhead (i.e., guard band). See 4.4--Volume partitioning for size limitations. | 4 |
