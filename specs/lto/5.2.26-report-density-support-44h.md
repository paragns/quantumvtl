# 5.2.26 REPORT DENSITY SUPPORT - 44h

The Report Density Support command is defined in SSC (see the latest version at
http://www.t10.org/drafts.htm#SCSI3_CMNDSETS). This clause decribes the specific implementation.

```
                                  Table 108 — REPORT DENSITY SUPPORT CDB

                                                                   Bit
         Byte
                     7 msb         6           5            4              3        2            1         0 lsb
           0                                            OPERATION CODE (44h)

           1                                            Reserved                                           MEDIA

           2
                                                                Reserved
           6
           7         MSB
                                                         ALLOCATION LENGTH
           8                                                                                                LSB
           9                                          Control Byte (see 5.1.2.3)
```

The following parameters apply:
- **MEDIA**:

| Value | Description |
|-------|-------------|
| 0b | report all supported densities for all supported medium types |
| 1b | report all supported densities for the current medium in the drive. If the device is not ready, the drive will return 2/0400h (NOT READY, NOT READY CAUSE NOT REPORTABLE). |

- **ALLOCATION LENGTH**: The maximum number of bytes to be transferred.

## 5.2.26.1 Report Density Support data layout

### 5.2.26.1.1 Density descriptor overview

Density descriptors contain information that may be returned by the drive. While the information is reported in a
manner useful to applications, SSC specifies fields that have been found to not be useful by applications in
selecting which density to use. Instead applications have been found to ignore the BITS PER MM field and the
TRACKS field. This device does not change the values returned in these fields by the format type -- media type
combinations and instead returns the largest value for the set of supported format type media type combinations.
The CAPACITY field is used by applications and does report values based on the format type -- media type
combinations. The DENSITY CODE fields, the ASSIGNING ORGANIZATION field, the DENSITY NAME field, and the
DESCRIPTION field are constant per density and are correctly reported.

This device may return different combinations of the descriptors in the REPORT DENSITY SUPPORT data
depending on drive generation, medium loaded, setting of the MEDIA bit, and settings of various mode
parameters related to densities.
In LTO-8 drives, if the MEDIA bit is set to one and a volume is mounted, the REPORT DENSITY SUPPORT data
may contain a number of different descriptors due to support of creating an Ultrium 7 Type M cartridge (i.e., M8)
out of an Ultrium 7 cartridge. The potential combinations are:
- L7 volume:
    a single descriptor indicating the "U-732 " density (i.e., 5Ch) (see table 112) with the WRTOK bit set to 1b
    and the DEFLT bit set to 1b, if the volume is an Ultrium 7 Type A cartridge (see 2.1.254) and:
    - the cartridge is not Type M eligible (see 2.1.256); or
    - the Preferred Cartridge Type for an Ultrium 7 volume is Type A cartridge;
- M8 capable Ultrium 7 cartridge; drive configured by library to make L7 volume:
    two descriptors, the first indicating the "U-732 " density (i.e., 5Ch) with the WRTOK bit set to 1b and the
    DEFLT bit set to 1b, and the second indicating the "U-832M " density (i.e., 5Dh) (see table 112) with the
    WRTOK bit set to 1b and the DEFLT bit set to 0b, if:
    - the volume is an Ultrium 7 Type A cartridge;
    - the cartridge is Type M eligible (see 2.1.256); and
    - the Preferred Cartridge Type for an Ultrium 7 volume is Type A cartridge;
- M8 capable Ultrium 7 cartridge; drive configured by library to make M8 volume:
    two descriptors, the first indicating the "U-732 " density (i.e., 5Ch) with the WRTOK bit set to 1b and the
    DEFLT bit set to 0b, and the second indicating the "U-832M " density (i.e., 5Dh) (see table 112) with the
    WRTOK bit set to 1b and the DEFLT bit set to 1b, if:
    - the volume is an Ultrium 7 Type A cartridge;
    - the cartridge is Type M eligible (see 2.1.256); and
    - the Preferred Cartridge Type for an Ultrium 7 volume is Type M cartridge (see 2.1.255);
- M8 volume:
    a single descriptor indicating the "U-832M " density (i.e., 5Dh) with the WRTOK bit set to 1b and the DEFLT
    bit set to 1b, if the volume is a Type M cartridge;
    or
- L8 volume:
    a single descriptor indicating the "U-832 " density (i.e., 5Eh) (see table 112) with the WRTOK bit set to 1b
    and the DEFLT bit set to 1b, if the volume is an Ultrium 8 Type A cartridge (see 2.1.254).

In other generation drives, there is one descriptor per generation cartridge.
