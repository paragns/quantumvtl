# 6.4.15 LP 18h: Protocol-specific port

See 5.2.8 -- LOG SELECT - 4Ch and see 5.2.9 -- LOG SENSE - 4Dh for directions on how to use this page. The Protocol-specific log page for SAS is used to report errors that have occurred on the phys of SAS drives.

There is one copy of this page for each port. This page is defined for SAS-attached devices only.

## 6.4.15.1 Parameter Reset Behavior (18h)

> Log pages may have different reset behaviors. Care should be taken to ensure counters are processed appropriately, including consideration of their reset behaviors. While reset behaviors may be different between products, an attempt is made to maintain compatibility between generations of a given product. See 6.4 -- Log Parameters (LP)

Specific flags may be cleared when corrective actions have removed the condition that caused the flag to be set.

## 6.4.15.2 Parameter Definitions (18h)

Table 280 defines the layout for the Protocol-Specific log parameter for SAS.

> The parameters in log pages should be dynamically parsed as some parameters may not be present and new parameters may be inserted. The relative location of parameters have changed and are anticipated to continue to change.

- **Bytes 0-1:** PARAMETER CODE: The relative target port identifier of the SSP target port that the log parameter describes.
- **Byte 2:** Parameter control byte
  - Bit 7: DU (0b)
  - Bit 6: Obsolete
  - Bit 5: TSD (0b)
  - Bit 4: ETC (0b)
  - Bits 3-2: TMC (00b)
  - Bits 1-0: FORMAT AND LINKING (11b)
- **Byte 3:** PARAMETER LENGTH: 38h
- **Byte 4:**
  - Bits 7-4: Reserved
  - Bits 3-0: PROTOCOL IDENTIFIER: 6h
- **Byte 5:** Reserved
- **Byte 6:** GENERATION CODE: 00h. Related to the Phy Control and Discover mode page which is not supported.
- **Byte 7:** NUMBER OF PHYS: 01h. The number of phys in the SAS target port (not in the entire SAS target device) and indicates the number of SAS phy log descriptors in the SAS phy log descriptor list.
- **Byte 8:** Reserved
- **Byte 9:** PHY IDENTIFIER: The phy identifier of the phy for which information is being returned.
- **Byte 10:** Reserved
- **Byte 11:** SAS PHY LOG DESCRIPTOR LENGTH: The number of bytes that follow in the SAS phy log descriptor. A SAS PHY LOG DESCRIPTOR LENGTH field set to 00h indicates that there are 44 additional bytes.
- **Byte 12:**
  - Bit 7: Reserved
  - Bits 6-4 -- **ATTACHED DEVICE TYPE:** The device type attached to this phy.

    | Code | Description |
    |------|-------------|
    | 000b | No device attached |
    | 001b | SAS device |
    | 010b | Edge Expander device |
    | 011b | Fan-out expander device |
    | All others | Reserved |

  - Bits 3-0 -- **ATTACHED REASON:** If a SAS phy or expander phy is attached, then the value of the REASON field received in the IDENTIFY address frame during the identification sequence.
- **Byte 13:**
  - Bits 7-4 -- **REASON:**

    | Code | Description |
    |------|-------------|
    | 0h | Unknown reason |
    | 1h | Power on |
    | 2h | Hard reset |
    | 3h | SMP PHY CONTROL function LINK RESET |
    | 4h | Loss of dword synchronization |
    | 5h | Obsolete |
    | 6h | Not supported |
    | 7h | Break Timeout Timer expired |
    | 8h | Phy test function stopped |
    | 9h | Expander device reduced functionality |
    | Ah-Fh | Reserved |

  - Bits 3-0 -- **NEGOTIATED LOGICAL LINK RATE:** The logical link rate being used by the phy (see table 251). This is negotiated during the link reset sequence.
- **Byte 14:**
  - Bits 7-4: Reserved
  - Bit 3: ATTACHED SSP INITIATOR PORT (0 = not attached, 1 = attached)
  - Bit 2: ATTACHED STP INITIATOR PORT (0 = not attached, 1 = attached)
  - Bit 1: ATTACHED SMP INITIATOR PORT (0 = not attached, 1 = attached. May be set to one for an attached expander device.)
  - Bit 0: Reserved
- **Byte 15:**
  - Bits 7-4: Reserved
  - Bit 3: ATTACHED SSP TARGET PORT (0 = not attached, 1 = attached)
  - Bit 2: ATTACHED STP TARGET PORT (0 = not attached, 1 = attached)
  - Bit 1: ATTACHED SMP TARGET PORT (0 = not attached, 1 = attached. Set to one for an attached expander device.)
  - Bit 0: Reserved
- **Bytes 16-23:** SAS ADDRESS: The port identifier of this SAS port.
- **Bytes 24-31:** ATTACHED SAS ADDRESS: The SAS ADDRESS of the attached port.
- **Byte 32:** ATTACHED PHY IDENTIFIER: The PHY IDENTIFIER of the attached port.
- **Bytes 33-39:** Reserved
- **Bytes 40-43:** INVALID DWORD COUNT: The number of invalid dwords that have been received outside of phy reset sequences.
- **Bytes 44-47:** RUNNING DISPARITY ERROR COUNT: The number of dwords containing running disparity errors that have been received outside of phy reset sequences.
- **Bytes 48-51:** LOSS OF DWORD SYNCHRONIZATION COUNT: The number of times the phy has restarted the link reset sequence because it lost dword synchronization.
- **Bytes 52-55:** PHY RESET PROBLEM COUNT: The number of times a phy reset problem occurred.
- **Bytes 56-57:** Reserved
- **Byte 58:** PHY EVENT DESCRIPTOR LENGTH: 00h. This drive does not return phy event descriptors.
- **Byte 59:** NUMBER OF PHY EVENT DESCRIPTORS: 00h. This drive does not return phy event descriptors.
