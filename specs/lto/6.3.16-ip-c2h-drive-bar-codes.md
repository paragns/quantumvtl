# 6.3.16 IP C2h: Drive Bar Codes

- **INQUIRY - 12h** (see 5.2.5 on page 84) describes how to request this page.

## 6.3.16.1 Returned Data - IP C2h: Drive Bar Codes

For LUN 0, the following data is returned:

> WARNING: This page should be dynamically parsed. Additional bar codes (i.e., fields) may be added in the future.

```
Table 223 — IP C2h: Drive Bar codes

                                                Bit
Byte    7           6           5           4           3           2           1       0
  0     PERIPHERAL QUALIFIER (000b)                     PERIPHERAL DEVICE TYPE (01h)
  1                                 PAGE CODE (C2h)
  2     (MSB)
                                PAGE LENGTH (n-3)
  3                                                                                     (LSB)
                            Bar code descriptor list
  4
                        Bar code descriptor (first)
                                    ...
                        Bar code descriptor (last)
  n
```

- **Byte 0** - Peripheral Data:
  - Bits 7-5: PERIPHERAL QUALIFIER: 000b
  - Bits 4-0: PERIPHERAL DEVICE TYPE: 01h
- **Byte 1** - PAGE CODE: C2h
- **Bytes 2-3** - PAGE LENGTH
- **Bytes 4-n** - Bar code descriptor list: Each bar code descriptor (see 6.3.16.1.1) describes a bar code returned by the drive.

### 6.3.16.1.1 Bar code descriptor

Descriptor that describes a bar code.

```
Table 224 — Bar code descriptor

                                                Bit
Byte    7           6           5           4           3           2           1       0
  0                         BAR CODE DESIGNATOR
  1                               Reserved
  2                       DATA IDENTIFIER LENGTH (n)
  3                       BAR CODE VALUE LENGTH (m-3)
  4
  4+(n-1)
                            BAR CODE VALUE
  4+n
  m
```

- **Byte 0** - BAR CODE DESIGNATOR: A value that designates which bar code is contained in the BAR CODE VALUE field.

| Value | Bar code |
|-------|---------|
| 00h | 11S Bar code from the drive brick (see 6.3.16.1.1.1) |

- **Byte 1** - Reserved
- **Byte 2** - DATA IDENTIFIER LENGTH: Length of the DATA IDENTIFIER field of the bar code value.
- **Byte 3** - BAR CODE VALUE LENGTH: Number of bytes to follow.
- **Bytes 4-m** - BAR CODE VALUE: Value that is returned when the bar code label is read by a bar code scanner. The first DATA IDENTIFIER LENGTH (n) bytes are a prefix used in a bar code to identify how the rest of the bar code is to be interpreted.

#### 6.3.16.1.1.1 00h -- 11S Bar code descriptor

The bar code value returned from a bar code scanner when reading the 11S bar code from the drive brick label.

```
Table 225 — 11S Bar code descriptor

                                                Bit
Byte    7           6           5           4           3           2           1       0
  0                     BAR CODE DESIGNATOR (00h)
  1                               Reserved
  2                     DATA IDENTIFIER LENGTH (03h)
  3                     BAR CODE VALUE LENGTH (16h)
  4
                        DATA IDENTIFIER ("11S")
  6
  7
                        DRIVE BRICK PART NUMBER
  13
  14
                       DRIVE BRICK SERIAL NUMBER
  25
```

- **Byte 0** - BAR CODE DESIGNATOR: 00h
- **Byte 1** - Reserved
- **Byte 2** - DATA IDENTIFIER LENGTH: 03h
- **Byte 3** - BAR CODE VALUE LENGTH: 16h
- **Bytes 4-6** - DATA IDENTIFIER: "11S" -- The identifier that indicates the (concatenated) IBM part number (7 characters) and serial number (12 characters).
- **Bytes 7-13** - DRIVE BRICK PART NUMBER: This is the drive brick part number set at time of manufacture, right-justified with leading zeros, in ASCII.
- **Bytes 14-25** - DRIVE BRICK SERIAL NUMBER: This is the drive brick serial number set at time of manufacture, right-justified with leading zeros, in ASCII.
