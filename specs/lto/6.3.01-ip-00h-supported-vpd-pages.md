# 6.3 Inquiry Vital Product Data Parameters (IP)

Inquiry vital product data parameters are returned to the Inquiry command. INQUIRY - 12h (see 5.2.5 on page 84) describes how to request these pages.

# 6.3.1 IP 00h: Supported Vital Product Data Pages

- **INQUIRY - 12h** (see 5.2.5 on page 84) describes how to request this page.

## 6.3.1.1 Returned Data - Inquiry Page 00h: Supported Inquiry Pages

The Supported Vital Product Data Page contains pages that the device will return.

```
Table 198 — IP 00h Supported Vital Product Data Inquiry Page

                                        Bit
Byte    7           6           5       4           3           2           1       0
  0     Peripheral Qualifier (000b)                 Peripheral Device Type (01h)
  1                             Page Code (00h)
  2                               Reserved
  3                           Page Length (n-3)
                          Supported VPD page list
  4                         VPD Page Code [first]
                                    ...
  n                         VPD Page Code [last]
```

- **Byte 0** - Peripheral Data:
  - Bits 7-5: Peripheral Qualifier: 000b
  - Bits 4-0: Peripheral Device Type: 01h
- **Byte 1** - Page Code: 00h
- **Byte 2** - Reserved
- **Byte 3** - Page Length (n-3)
- **Bytes 4-n** - Supported VPD pages: This field is a list of 1-byte long page codes and may include some or all of the following:

| Code | Inquiry Page |
|------|-------------|
| 00h | IP 00h: Supported Vital Product Data Pages (see 6.3.1 on page 227) |
| 03h | IP 03h: Firmware Designation (see 6.3.2 on page 228) |
| 80h | IP 80h: Unit Serial Number (see 6.3.3 on page 230) |
| 83h | IP 83h: Device Identification (see 6.3.4 on page 230) |
| 86h | IP 86h: Extended INQUIRY Data (see 6.3.5 on page 236) |
| 87h | IP 87h: Mode Page Policy (see 6.3.6 on page 238) |
| 88h | IP 88h: SCSI ports (see 6.3.7 on page 240) |
| 90h | IP 90h: Protocol-Specific Logical Unit Information (see 6.3.8 on page 242) |
| B0h | IP B0h: Sequential-Access device capabilities (see 6.3.9 on page 243) |
| B1h | IP B1h: Manufacturer-assigned Serial Number (see 6.3.10 on page 244) |
| B3h | IP B3h: Automation Device Serial Number (see 6.3.11 on page 245) |
| B4h | IP B4h: Data Transfer Device Element Address (see 6.3.12 on page 245) |
| B5h | IP B5h: Logical Block Protection (see 6.3.13 on page 246) |
| C0h | IP C0h: Drive Component Revision Levels (see 6.3.14 on page 248) |
| C1h | IP C1h: Drive Serial Numbers (see 6.3.15 on page 250) |
| C2h | IP C2h: Drive Bar codes (see 6.3.16 on page 251) |
| C3h | IP C3h: Subcomponent Version List (see 6.3.17 on page 253) |
| C7h | IP C7h: Device Unique Configuration Data (see 6.3.18 on page 254) |
| C8h | IP C8h: Mode Parameter Default Settings (see 6.3.19 on page 254) |
| E0-EF | Vendor-Reserved (Attachment Specification Information) |

> NOTE 36 - Only those drives that have the AS/400 (iSeries) attachment enabled will have valid data in pages E0h/E1h. The contents of pages E0h/E1h are not specified in this document.

> NOTE 37 - The drive may report other pages in the supported VPD page list or the drive may support and return pages not specified in this document.
