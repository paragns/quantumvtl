# 5.2.12 PERSISTENT RESERVE IN (PRIN)- 5Eh

The PERSISTENT RESERVE IN command is defined in SPC (see the latest version at
http://www.t10.org/drafts.htm#SCSI3_CMNDSETS). This clause decribes the specific implementation.

```
                              Table 59 — PERSISTENT RESERVE IN CDB

                                                        Bit
     Byte
               7 msb        6          5          4              3          2          1       0 lsb
      0                                           OPCODE (5Eh)

      1                  Reserved                                     SERVICE ACTION

      2
                                                      Reserved
      5
      6        (MSB)
                                                ALLOCATION LENGTH
      8                                                                                        (LSB)
      9                                      Control Byte (see 5.1.2.3)
```

The following parameters apply:

- **SERVICE ACTION**:

  | Value | Description |
  |-------|-------------|
  | 00h | READ KEYS - Reads all registered Reservation Keys (see 5.2.12.1) |
  | 01h | READ RESERVATION - Reads all current persistent reservations (see 5.2.12.2) |
  | 02h | REPORT CAPABILITIES - Returns capability information (see 5.2.12.3) |
  | 03h | READ FULL STATUS - Reads complete information about all registrations and the persistent reservations, if any (see 5.2.12.4) |

- **ALLOCATION LENGTH**: The maximum number of bytes to be transferred.


## 5.2.12.1 READ KEYS service action

The READ KEYS service action requests that the device server return a parameter list containing a header and a
list of each currently registered I_T nexus' reservation key.

The layout for the parameter data provided in response to a PERSISTENT RESERVE IN command with the
READ KEYS service action is shown in table 60.

```
                  Table 60 — PERSISTENT RESERVE IN parameter data for READ KEYS

                                                           Bit
   Byte
              7 msb         6           5            4              3        2          1     0 lsb
    0         (MSB)
                                                     PRGENERATION
    3                                                                                         (LSB)
    4         (MSB)
                                                 ADDITIONAL LENGTH (n-7)
    7                                                                                         (LSB)
                                                   Reservation key list
    8         (MSB)
                                                  Reservation key [first]
   15                                                                                         (LSB)



                                                           ...
   n-7        (MSB)
                                                  Reservation key [last]
    n                                                                                         (LSB)
```

| Byte | Description |
|------|-------------|
| 0-3 | PRGENERATION: Counter for Persistent Reserve Out Command requests |
| 4-7 | ADDITIONAL LENGTH: A count of the number of bytes in the Reservation key list |
| 8-15 | First Reservation Key |
| 16-n | Additional Reservation keys: additional reservation keys are not supported. |


## 5.2.12.2 READ RESERVATION service action

The READ RESERVATION service action requests that the device server return a parameter list containing a
header and the persistent reservation, if any, that is present in the device server.

The layout for the parameter data provided in response to a PERSISTENT RESERVE IN command with the
READ RESERVATION service action is shown in table 61.

```
               Table 61 — PERSISTENT RESERVE IN parameter data for READ RESERVATION

                                                             Bit
  Byte
               7 msb        6            5             4              3            2           1        0 lsb
    0          (MSB)
                                                       PRGENERATION
    3                                                                                                   (LSB)
    4          (MSB)
                                                     ADDITIONAL LENGTH
    7                                                                                                   (LSB)
                             If reservation exists (i.e., ADDITIONAL LENGTH > 0000 0000h)
    8          (MSB)
                                                      RESERVATION KEY
   15                                                                                                   (LSB)
   16
                                                           Obsolete
   19
   20                                                      Reserved
   21                            SCOPE                                                  TYPE

   22
                                                           Obsolete
   23
```

| Byte | Description |
|------|-------------|
| 0-3 | PRGENERATION: Counter for Persistent Reserve Out Command requests |
| 4-7 | ADDITIONAL LENGTH: A count of the number of bytes in the Reservation key list. |
|     | Value `00000000h`: No reservation exists |
|     | Value `00000010h`: Reservation exists and this much data follows |
| 8-n | Reservation descriptor, if any: (defined below) |
| 8-15 | Reservation Key |
| 16-19 | Obsolete |
| 20 | Reserved |
| 21 | Bit 7-4: SCOPE: persistent reservation applies to the entire logical unit: 0h |
|    | Bit 4-0: TYPE: |
|    | Value `3h`: Exclusive Access |
|    | Value `6h`: Exclusive Access -- Registrants only |
|    | Value `8h`: Exclusive Access -- All Registrants |
|    | Value `others`: Not supported |
| 22-23 | Obsolete |


## 5.2.12.3 REPORT CAPABILITIES service action

The REPORT CAPABILITIES service action requests that the device server return information on persistent
reservation features.

The layout for the parameter data provided in response to a PERSISTENT RESERVE IN command with the
REPORT CAPABILITIES service action is shown in table 62.

```
          Table 62 — PERSISTENT RESERVE IN parameter data for REPORT CAPABILITIES

                                                               Bit
Byte
         7 msb             6            5               4               3        2         1            0 lsb
  0       (MSB)
                                                        LENGTH (0008h)
  1                                                                                                    (LSB)
  2                    Reserved                        CRH           SIP_C      ATP_C   Reserved       PTPL_C

  3        TMV                 ALLOW COMMANDS (000b)                         Reserved                  PTPL_A

                                             Persistent Reservation Type Mask
  4     WR_EX_AR      EX_AC_RO      WR_EX_RO       Rsvd              EX_AC      Rsvd     WR_EX          Rsvd
  5                                              Reserved                                             EX_AC_AR

  6
                                                             Reserved
  7
```

| Byte | Description |
|------|-------------|
| 0-1 | LENGTH: 0008h |
| 2 | Bit 7-5: Reserved |
|   | Bit 4: CRH (Compatible Reservation Handling): 1b |
|   | Bit 3: SIP_C (Specify Initiator Ports Capable): 0b |
|   | Bit 2: ATP_C (All Target Ports Capable): 0b |
|   | Bit 1: Reserved |
|   | Bit 0: PTPL_C (Persist Through Power Loss Capable): 0b |
| 3 | Bit 7: TMV (Type Mask Valid): 1b |
|   | Bit 6-4: ALLOW COMMANDS: 000b |
|   | Value `000b`: No information is provided about whether certain commands are allowed through certain types of persistent reservations. |
|   | Value `001b-100b`: Not Supported. TUR is allowed through Exclusive Access reservations [and info about certain other commands and Write Exclusive reservations]. |
|   | Value `101b-111b`: Reserved |
|   | Bit 3-1: Reserved |
|   | Bit 0: PTPL_A (Persist Through Power Loss Activated): 0b |
| 4 | Persistent Reservation Type Mask (byte 4) |
|   | Bit 7: WR_EX_AR (Write Exclusive -- All Registrants): 0b |
|   | Bit 6: EX_AC_RO (Exclusive Access -- Registrants Only): 1b |
|   | Bit 5: WR_EX_RO (Write Exclusive -- Registrants Only): 0b |
|   | Bit 4: Reserved |
|   | Bit 3: EX_AC (Exclusive Access): 1b |
|   | Bit 2: Reserved |
|   | Bit 1: WR_EX (Write Exclusive): 0b |
|   | Bit 0: Reserved |
| 5 | Persistent Reservation Type Mask (byte 5) |
|   | Bit 7-1: Reserved |
|   | Bit 0: EX_AC_AR (Exclusive Access -- All Registrants): 0b |
| 6-7 | Reserved |


## 5.2.12.4 READ FULL STATUS service action

The READ FULL STATUS service action requests that the device server return a parameter list describing the
registration and persistent reservation status of each currently registered I_T nexus for the logical unit.

The layout for the parameter data provided in response to a PERSISTENT RESERVE IN command with the
READ FULL STATUS service action is shown in table 63.

```
             Table 63 — PERSISTENT RESERVE IN parameter data for READ FULL STATUS

                                                              Bit
  Byte
             7 msb          6            5             4              3              2      1        0 lsb
   0         (MSB)
                                                       PRGENERATION
   3                                                                                                 (LSB)
   4         (MSB)
                                                   ADDITIONAL LENGTH (n-7)
   7                                                                                                 (LSB)
                                                    Full status descriptors
   8
                                         Full status descriptor [first] (see table 64)
                                                              ...



                                          Full status descriptor [last] (see table 64)
   n
```

| Byte | Description |
|------|-------------|
| 0-3 | PRGENERATION: |
| 4-7 | ADDITIONAL LENGTH (n-7) |
| 8-n | Full status descriptors (see 5.2.12.4.1). |


### 5.2.12.4.1 Full status descriptors

The layout of the full status descriptors is shown in table 64. Each full status descriptor describes one or more
registered I_T nexuses. The device returns persistent reservations status information for every registered I_T
nexus.

```

   12                                          Reserved                                     ALL_TG_PT   R_HOLDER

   13                              SCOPE                                                 TYPE

   14
                                                              Reserved

                                                 RELATIVE TARGET PORT IDENTIFIER

                                               ADDITIONAL DESCRIPTOR LENGTH (n-23)

```

| Byte | Description |
|------|-------------|
| 0-7 | RESERVATION KEY: |
