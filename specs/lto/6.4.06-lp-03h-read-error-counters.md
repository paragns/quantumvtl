# 6.4.6 LP 03h: Read Error Counters

See 5.2.8 -- LOG SELECT - 4Ch and see 5.2.9 -- LOG SENSE - 4Dh for directions on how to use this page. This page is for Read Errors.

## 6.4.6.1 Parameter Definitions (03h)

Parameters 0000h through 0002h are not supported and are returned as 0000_0000h.

> Log pages may have different reset behaviors. Care should be taken to ensure counters are processed appropriately, including consideration of their reset behaviors. While reset behaviors may be different between products, an attempt is made to maintain compatibility between generations of a given product. See 6.4 -- Log Parameters (LP)

> The parameters in log pages should be dynamically parsed as some parameters may not be present and new parameters may be inserted. The relative location of parameters have changed and are anticipated to continue to change.

| Parameter Code | Counter: Description | Size |
|---------------|----------------------|------|
| 0000h | Not supported | 4 |
| 0001h | Not supported | 4 |
| 0002h | **Total Read Errors {LP03h:0002h}:** This is the sum of parameter 0003h and 0006h | 4 |
| 0003h | **Total Corrected Read Errors {LP03h:0003h}:** ECC C2 correction was used. It is incremented on the initial read whether it triggered error recovery (temp or perm) or not. This is a roll-up of LP 34h:0000h (see 6.4.23). | 4 |
| 0004h | Not Supported | 4 |
| 0005h | **Total Read Kibibytes Processed {LP03h:0005h}:** Each count represents a kibibyte (1024 bytes) processed across the host interface during read-type commands. The count does not include ERP retries. This field is identical to the Host Read Kibibytes Processed field of Page Code 38h, parameter code 0003h (see 6.4.25). | 6 |
| 0006h | **Total Uncorrected Read Errors {LP03h:0006h}:** Count of Temp errors where ECC data correction was insufficient to nominally read the dataset. This count is independent of whether or not ECC C2 correction occurred on the initial error that triggered successful error recovery, but it excludes temps which started with servo errors. This count is updated at the end of a successful data recovery. This is a roll-up of LP 34:0005h, 34:0007h (see 6.4.23). | 4 |
| 8000h | Unspecified | 8 |
