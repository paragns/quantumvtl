# 6.3.6 IP 87h: Mode Page Policy

The Mode Page Policy Page (see table 208) indicates which mode page policy is in effect for each mode page supported by the logical unit.

- **INQUIRY - 12h** (see 5.2.5 on page 84) describes how to request this page.

## 6.3.6.1 Returned Data - IP 87h: Mode Page Policy

The data returned is shown in table 208.

```
Table 208 — IP 87h Mode Page Policy page

                                                    Bit
Byte    7           6           5           4           3           2           1       0
  0     Peripheral Qualifier (000b)                     Peripheral Device Type (01h)
  1                                 Page Code (87h)
  2     (MSB)
                                Page Length (n-3)
  3                                                                                     (LSB)
                            Mode page policy descriptor list
  4
            Mode page policy descriptor (first)
  7

                                    ...
  n-3
            Mode page policy descriptor (last)
  n
```

- **Byte 0** - Peripheral Data:
  - Bits 7-5: Peripheral Qualifier: 000b
  - Bits 4-0: Peripheral Device Type: 01h
- **Byte 1** - Page Code: 87h
- **Bytes 2-3** - Page Length (n-3)
- **Bytes 4-n** - Mode page policy descriptor list. Each mode page policy descriptor (see 6.3.6.1.1) contains information describing the mode page policy for one or more mode pages or subpages. The information in the mode page policy descriptors in this page describe the mode page policy for every mode page and subpage supported by the logical unit.

  The first mode page policy descriptor in the list contains a POLICY PAGE CODE field set to 3Fh and a POLICY SUBPAGE CODE field set to FFh, indicating that the descriptor applies to all mode pages and subpages not described by other mode page policy descriptors. The POLICY PAGE CODE field is set to 3Fh and the POLICY SUBPAGE CODE field is set to FFh only in the first mode page policy descriptor in the list. If the POLICY PAGE CODE field contains a value other than 3Fh or a POLICY SUBPAGE CODE field contains a value other than FFh, then the POLICY PAGE CODE field and the POLICY SUBPAGE CODE field indicate a single mode page and a subpage to which the descriptor applies.

### 6.3.6.1.1 Mode page policy descriptor

```
Table 209 — Mode page policy descriptor of IP 87h

                                                    Bit
Byte    7           6           5           4           3           2           1       0
  0         Reserved                                    POLICY PAGE CODE
  1                             POLICY SUBPAGE CODE
  2     MLUS                    Reserved                                    MODE PAGE POLICY
  3                                 Reserved
```

- **Byte 0**:
  - Bits 7-6: Reserved
  - Bits 5-0: POLICY PAGE CODE: The mode page to which the descriptor applies.
- **Byte 1** - POLICY SUBPAGE CODE: Indicates the subpage to which the descriptor applies.
- **Byte 2**:
  - Bit 7: MLUS (multiple logical units share): Indicates if the mode page is shared by multiple logical units.

    | Value | Description |
    |-------|------------|
    | 0b | Indicates that the mode page and subpage identified by the POLICY PAGE CODE field and POLICY SUBPAGE CODE field is not shared by more than one logical unit. |
    | 1b | Indicates that the mode page and subpage identified by the POLICY PAGE CODE field and POLICY SUBPAGE CODE field may be shared by more than one logical unit. |

  - Bits 6-2: Reserved
  - Bits 1-0: MODE PAGE POLICY: Indicates the mode page policy (see 4.7.1) for the mode page and subpage identified by the POLICY PAGE CODE field and POLICY SUBPAGE CODE field.

    | Code | Description |
    |------|------------|
    | 00b | \<Shared\> |
    | 01b | \<Per target port\> |
    | 10b | Obsolete |
    | 11b | \<Per I_T nexus\> |

The mode page policies including the multiple logical unit shares are described in 4.7.1.2--Mode page policy.
