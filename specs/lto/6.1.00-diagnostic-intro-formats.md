# 6.1 Diagnostic Parameters (Diag)

Diagnostic parameters are used with the SEND DIAGNOSTIC command. See 5.2.36--SEND DIAGNOSTIC - 1Dh and 5.2.23--RECEIVE DIAGNOSTIC RESULTS - 1Ch for additional information on the commands.

Terms used in this clause are:

| Term     | Description                |
|----------|----------------------------|
| CD       | Cryptographic Diagnostic   |
| Diag     | Diagnostic                 |
| Page     | Page Code                  |
| Parm     | Parameter                  |
| RcvDiag  | Receive Diagnostic Results |
| SendDiag | Send Diagnostic            |

## 6.1.1 Diag Page Formats

The drive supports two diagnostic pages for normal use: Page Code 00h and 80h. Page Code 81h is for engineering use only.

### 6.1.1.4 Page 80h

Page Code 80h is a general purpose page for sending flags and diagnostic parameters to the target.

#### 6.1.1.4.1 SendDiag - Page 80h

The layout for the Send Diagnostic command is shown in table 157.

```
SendDiag - Page 80h Parm Data (Table 157)

  Byte    7     6     5     4     3     2     1     0
    0                    PAGE CODE (80h)
    1                       Reserved
    2   (MSB)
                     PAGE LENGTH (n-3)
    3                                              (LSB)
    4
                        DIAGNOSTIC ID
    5
    6                      FLAGS                    CR
    7                       Reserved
    8
                     Diagnostic Parameters
    n
```

- **Page Code** (byte 0): 80h
- **Reserved** (byte 1)
- **Page Length** (bytes 2-3): n-3
- **Diagnostic ID** (bytes 4-5): This field specifies the diagnostic that is to be run
- **Flags/CR** (byte 6):
  - Bits 7-1: FLAGS
  - Bit 0: CR (cartridge required) -- Set to 1b when a cartridge is required for a diagnostic. When 1b, a cartridge must be loaded at the load point and ready for the SEND DIAGNOSTIC command to be accepted. See specific diagnostic descriptions for cartridge use: some diagnostics require this bit to be set to 1b, some require it to be set to 0b, and some do not require a specific bit setting.
    - 0b: No cartridge required
    - 1b: Cartridge required
- **Reserved** (byte 7)
- **Diagnostic Parameters** (bytes 8-n): The Diagnostic Parameters field contains the parameters required to run the diagnostic.

See 6.1.2 for a list of supported diagnostic page 80h routines.

- **Page Code** (byte 0): 80h
- **Reserved** (byte 1)

- **FLAGS** (byte 6)

See the individual Send Diagnostic parameter descriptions for the field contents. See 6.1.2--Supported Page 80h Diags for a list of diagnostic parameters supported by the drive.

#### 6.1.1.5.1 RcvDiag - Page 80h Typical Results

The typical results for diagnostics that generate results data is shown in table 159.

```
RcvDiag Data - Page 80h Results (Table 159)

  Byte   7 msb   6     5     4     3     2     1   0 lsb
    0                    PAGE CODE (80h)
    1                       Reserved
    2   (MSB)
                   PAGE LENGTH (004Dh)
    3                                              (LSB)
    4   (MSB)
                       DIAGNOSTIC ID
    5                                              (LSB)
    6                       FLAGS
    7                       Reserved
                       Diagnostic Results
    8             Reserved            DIAG   SIM/MIM
                                     BLOCKED PRESENT  ERROR
    9
              SIM/MIM message or all zeros
   80
```

> Note 1 - The ERROR bit in byte 8 is set when the diagnostic detects an error.

> Note 2 - The SIM/MIM Present bit in byte 8 is set when a SIM or MIM message is contained in the diagnostic results.

> Note 3 - SIM/MIM messages are defined exactly as described in 6.1.1.5.2--SIM/MIM Message; the SIM/MIM is not valid if the SIM/MIM Present bit is 0b.

> Note 4 - The Diagnostic Blocked bit is set when the diagnostic cannot run all its tests. This occurs on some diagnostics if a cartridge is in the drive.

- **Page Code** (byte 0): 80h
- **Reserved** (byte 1)
- **Page Length** (bytes 2-3): 004Dh
- **Diagnostic ID** (bytes 4-5): This field contains the same value as that sent with the Send Diagnostic command for which this response is associated.
- **FLAGS** (byte 6)
- **Reserved** (byte 7)
- **Byte 8**:
  - Bits 7-3: Reserved
  - Bit 2: DIAGNOSTIC BLOCKED -- Indicates if the diagnostic could not run all its tests, which may occur on some diagnostics if a cartridge is loaded in the drive.
    - 0b: the diagnostic was not blocked from running
    - 1b: the diagnostic was blocked from running
  - Bit 1: SIM/MIM PRESENT:
    - 0b: a SIM or MIM message is not contained in the diagnostic results and the SIM/MIM field is set to all zeros.
    - 1b: a SIM or MIM message is contained in the diagnostic results
  - Bit 0: ERROR
- **SIM/MIM** (bytes 9-80): This field contains a SIM/MIM Message (see 6.1.1.5.2) or all zeros as indicated by the SIM/MIM PRESENT bit.

#### 6.1.1.5.2 SIM/MIM Message

SIMs (Service Information Messages), and MIMs (Medium Information Messages) provide the initiator and operator details on service problems encountered by the device. The first 9 bytes are common to both the SIMs and the MIMs.

##### 6.1.1.5.2.1 SIM/MIM Header Data

```
Diag SIM Data Structure (Table 160)

  Byte    7     6     5     4     3     2     1     0
    0                    Page Code (31h)
    1                       Reserved
    2   (MSB)
                     PAGE LENGTH (0044h)
    3                                              (LSB)
    4   (MSB)
                   PARAMETER CODE (0000h)
    5                                              (LSB)
    6              PARAMETER CONTROL (61h)
    7              PARAMETER LENGTH (40h)
    8                  SIM/MIM INDICATOR
    9
                      SIM or MIM data
   71
```

- **Page Code** (byte 0):
  - Bits 7-6: Reserved
  - Bits 5-0: Page Code: 31h
- **Reserved** (byte 1)
- **Page Length** (bytes 2-3): 0044h
- **Parameter Code** (bytes 4-5): 0000h
- **Parameter control byte** (byte 6):
  - Bit 7: DU (Disable Update): 0b
  - Bit 6: DS (Disable Save): 1b
  - Bit 5: TSD (Target Save Disable): 1b
  - Bit 4: ETC (Enable Threshold Comparison): 0b
  - Bits 3-2: TMC (Threshold Met Comparison): 00b
  - Bit 1: Reserved
  - Bit 0: LP (List Parameter): 1b
- **Parameter length** (byte 7): 40h
- **SIM/MIM Indicator** (byte 8):
  - 00h: Bytes 9 through 71 are invalid.
  - 01h: Bytes 9 through 71 are a SIM message. See 6.1.1.5.2.2--SIM Messages.
  - 02h: Bytes 9 through 71 are a MIM message. See 6.1.1.5.3--MIM Messages.
  - 03-FFh: Bytes 9 through 71 are invalid.

##### 6.1.1.5.2.2 SIM Messages

The following data are the parameters for the hardware SIM message.

```
Diag SIM Data Structure (Table 161)

  Byte    7     6     5     4     3     2     1     0
  9-15                     Reserved
 16-19            PRODUCT REVISION LEVEL
 20-21           SIM MESSAGE CODE (MSB/LSB)
 22-23                     Reserved
   24              EXCEPTION MESSAGE CODE
   25               SERVICE MESSAGE CODE
   26           SERVICE MESSAGE SEVERITY CODE
   27                      Reserved
 28-29            EXCEPTION DATA (MSB/LSB)
 30-33            SCD ERROR CODE (MSB/LSB)
 34-37               FIRST FSC (MSB/LSB)
 38-41                LAST FSC (MSB/LSB)
 42-45            PRODUCT ID (8000h) (MSB/LSB)
              Product Identifier:

 49-50           PLANT OF MANUFACTURE
   51                     '-'
 52-63            SERIAL NUMBER (MSB/LSB)

```

- **Reserved** (bytes 9-15)
- **Product Revision Level** (bytes 16-19): Microcode level. Same as bytes 32-35 in Standard Inquiry data.
- **SIM Message Code** (bytes 20-21):

  | Value (ASCII) | Description |
  |---------------|-------------|
  | '00' | No Message |
  | '41' | Device Degraded. Call for Service |
  | '42' | Device Hardware Failure. Call for Service |
  | '43' | Service Circuits Failed, Operations Not Affected. Call for Service |
  | '55' | Drive Needs Cleaning. Load Cleaning Cartridge |
  | '57' | Drive Has Been Cleaned |
  | All Others | Device Message |

- **Reserved** (bytes 22-23)
- **Exception Message Code** (byte 24):

  | Value (ASCII) | Description |
  |---------------|-------------|
  | '0' | Vendor-Reserved |
  | '1' | Effect of Failure Is Unknown |
  | '2' | Device Exception. No Performance Impact |
  | '3' | Exception on Host Interface xx (see bytes 28-29 Exception Data xx in this SIM record for the xx value) |
  | '4' | Device Exception on ACF |
  | '5' | Device Exception on Operator Panel |
  | '6' | Device Exception on Tape Path |
  | '7' | Device Exception in Drive |
  | '8' | Cleaning Required |
  | '9' | Cleaning Done |
  | 'A'-'F' | Vendor-Reserved |

- **Service Message Code** (byte 25):

  | Value (ASCII) | Description |
  |---------------|-------------|
  | '0' | Vendor-Reserved |
  | '1' | Repair Impact is Unknown |
  | '2'-'6' | Vendor-Reserved |
  | '7' | Repair Will Disable Access to Device serno (serno refers to serial number in bytes 52-63 of this SIM record) |
  | '9' | Clean Device |
  | 'A' | Device Cleaned |
  | 'B' | Device Cleaning For Performance Reasons is Required |
  | 'C'-'F' | Vendor-Reserved |

- **Service Message Severity Code** (byte 26):

  | Value (ASCII) | Description |
  |---------------|-------------|
  | '0' | SIM severity code "Service" |
  | '1' | SIM severity code "Moderate" |
  | '2' | SIM severity code "Serious" |
  | '3' | SIM severity code "Acute" |
  | '4'-'9', 'A'-'F' | Vendor-Reserved |

- **Reserved** (byte 27)
- **Exception Data** (bytes 28-29): Interface Data is returned when byte 24 (Exception Message Code) in this SIM record contains the ASCII value '3'.

  | Value (ASCII) | Description |
  |---------------|-------------|
  | '00' | interface 0 is indicated |
  | '01' | interface 1 is indicated |

- **SCD Error Code** (bytes 30-33): Error code indicated on SCD (ASCII code). For example 0x00000035 for SCD '5'.
- **First FSC** (bytes 34-37)
- **Last FSC** (bytes 38-41)
- **Product ID** (bytes 42-45): '8000' (these 4 bytes define "TAPE")
- **Product Identifier** (bytes 46-63): Broken out into constituent bytes:
  - Bytes 46-48: MANUFACTURER: "IBM"
  - Bytes 49-50: PLANT OF MANUFACTURE
  - Byte 51: '-' (Dash symbol)
  - Bytes 52-63: SERIAL NUMBER
- **Device Type** (bytes 64-71): Device type portion of PRODUCT IDENTIFICATION; Inquiry Standard Data bytes 16-23.

- **MIM Message Code** (bytes 20-21):

  | Value (ASCII) | Description |
  |---------------|-------------|

  | Value (ASCII) | Description |

  | '4' | Medium Degraded |
  | '6' | CM Error |
  | '7' | Medium Exception |
  | Others | Vendor-Reserved |

- **Media Message Code** (byte 25)
- **Media Message Severity Code** (byte 26):

  | Value (ASCII) | Description |
  |---------------|-------------|
  | '0' | Service |
  | '1' | Moderate-High Temp Read/Write Errors Detected |
  | '2' | Serious-Permanent Read/Write Errors Detected |
  | '3' | CM Error |
  | Others | Vendor-Reserved |

- **Reserved** (bytes 27-29)
- **Fault Symptom Code (FSC)** (bytes 30-33)
- **VOLID** (bytes 34-39): Volume Serial Number (in ASCII). Only valid if indicated by VOLID Valid Flag (byte 40).
- **VOLID Valid** (byte 40):

  | Value (ASCII) | Description |
  |---------------|-------------|
  | '0' | VOLID (bytes 34-39) not valid |
  | '1' | VOLID valid, obtained from tape |
  | '3' | VOLID valid, obtained from host (MAM Attribute ID 0806 in CM) |
  | '5' | VOLID valid, obtained from library |
  | Others | Vendor-Reserved for future use (odd number will always indicate VOLID valid) |

- **Reserved** (byte 41)
- **Product ID** (bytes 42-45): '8000' (these 4 bytes define "TAPE")
- **Product Identifier** (bytes 46-63): Broken out into constituent bytes:
  - Bytes 46-48: MANUFACTURER: "IBM"
  - Bytes 49-50: PLANT OF MANUFACTURE
  - Byte 51: '-' (Dash symbol)
  - Bytes 52-63: SERIAL NUMBER
- **Device Type** (bytes 64-71): Device type portion of PRODUCT IDENTIFICATION; Inquiry Standard Data bytes 16-23.
