# 6.1.15 Diag - 0175h: Use FMR Cartridge (not on LTO5)

See 5.2.36--SEND DIAGNOSTIC - 1Dh and 5.2.23--RECEIVE DIAGNOSTIC RESULTS - 1Ch for additional information on the commands. This diagnostic prepares the drive to receive a field microcode replacement (FMR) cartridge for the purpose of updating the functional microcode load of the drive.

This enables the drive to accept the mounting of an FMR cartridge and to automatically perform the processing that updates functional microcode with code read from that cartridge. This enablement is cleared:

1. after 60 seconds;
2. on completion of processing the FMR cartridge if that processing results in success;
3. on completion of processing the FMR cartridge if that processing results in failure; or
4. on the mounting of a non-FMR cartridge.

If there is a Data cartridge in the drive, mounted and ready when this diagnostic is run, then this diagnostic fails for an invalid image.

If an attempt is made to load a Data cartridge when the drive is prepared to receive an FMR cartridge (i.e., after successful completion of this diagnostic and before the enablement is cleared), then the cartridge is rejected for an unsupported format.

> NOTE 35 - This function is also available from maintenance mode and the library interface (i.e. LDI and ADI). The MUE bit of the Logical Unit subpage (see the ADI Implementation Reference) performs this same diagnostic.
