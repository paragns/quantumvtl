# 6.6 Mode Parameters (MP)

Mode parameters are used with the MODE SELECT (6/10) - 15h/55h (see 5.2.10) commands and the MODE
SENSE (6/10) - 1Ah/5Ah (see 5.2.11) commands. Mode Page Behaviors (see 4.7 on page 40) describes this
device's non-standard behaviors related to mode parameters.

## 6.6.1 Mode Parameter List for Mode Select (6/10)

For Mode Select (6) the mode pages are preceded by a 4-byte mode parameter header below, and an optional
8-byte block descriptor (see 6.6.2.2). Table 321 shows the layout of the mode parameter list for Mode Select (6).

```
Table 321 — Mode Parameter List for Mode Select (6)

                                                    Bit
Byte
         7 msb     6         5         4         3         2         1    0 lsb
  0
                                      Mode Parameter Header
  3
  4                                    Block Descriptor
4+n-1                 (n = 0 or n = 8; if n=0 Block Descriptor does not exist)

 4+n
                                        Mode Pages
  n
```

For Mode Select (10) the mode pages are preceded by a 8-byte mode parameter header (see 6.6.1.1) and an optional 8-byte block descriptor (see 6.6.1.2). Mode page descriptions begin at MP 01h: Read-Write Error Recovery on page 370.

```

## 6.6.1.1 Mode Parameter Header for Mode Select (6/10)

There is one copy of the mode parameter header for each initiator. Mode parameter header and block descriptor
policy (see 4.7.1.1 on page 40) describes this devices non-standard behavior related to fields in the header.

> Note that mounting a volume that modifies the value of fields in the mode parameter header does not establish a
> unit attention condition.

```
Table 323 — Mode Parameter Header for Mode Select (6)

                                                    Bit
Byte
          7 msb     6       5         4         3       2         1    0 lsb
  0                            Reserved (Mode Data Length)
  1                                  Medium Type
  2        WP           Buffer Mode                         Speed
  3                           Block Descriptor Length
```

```
Table 324 — Mode Parameter Header for Mode Select (10)

                                                    Bit
Byte
          7 msb     6       5         4         3       2         1    0 lsb
  0       MSB
                              Reserved (Mode Data Length)
  1                                                                    LSB
  2                                  Medium Type
  3        WP             BUFFERED MODE                         SPEED

  4
                                      Reserved
  5
  6       MSB
                               BLOCK DESCRIPTOR LENGTH
  7                                                                    LSB
```

Mode parameter header field descriptions follow:

- **Byte 0-1**: Reserved (Mode Data Length): must be zero.
- **Byte 2**: MEDIUM TYPE: (changeable-ignored)

  | Value | Description |
  |-------|-------------|
  | 00h | No volume is loaded |
  | 18h | Ultrium 1 Data volume is loaded |
  | 28h | Ultrium 2 Data volume is loaded |
  | 38h | Ultrium 3 Data volume is loaded |
  | 3Ch | Ultrium 3 WORM volume is loaded |
  | 48h | Ultrium 4 Data volume is loaded |
  | 4Ch | Ultrium 4 WORM volume is loaded |
  | 58h | Ultrium 5 Data volume is loaded |
  | 5Ch | Ultrium 5 WORM volume is loaded |
  | 68h | Ultrium 6 Data volume is loaded |
  | 6Ch | Ultrium 6 WORM volume is loaded |
  | 78h | Ultrium 7 Data volume is loaded |
  | 7Ch | Ultrium 7 WORM volume is loaded |
  | 88h | Ultrium 8 Data volume is loaded |
  | 8Ch | Ultrium 8 WORM volume is loaded |
  | 98h | Ultrium 9 Data volume is loaded |
  | 9Ch | Ultrium 9 WORM volume is loaded |

- **Byte 3**: Device-Specific Parameter - Sequential Access Devices
  - Bit 7: **WP**: (changeable-ignored)
  - Bits 6-4: **BUFFERED MODE**: 001b \<Shared\> (changeable)

    | Value | Description |
    |-------|-------------|
    | 000b | Good status is reported when data on medium |
    | 001b | Good status is reported when data is in buffer |
    | 010b | Not supported |
    | 011b - 111b | Reserved |

  - Bits 3-0: **SPEED**: 0h (use default read/write speed) \<Shared\> (changeable)
    - In this drive, contrary to the standards, speed 1h is the fastest speed and higher numbers are progressively slower speeds.

    > **WARNING**: Setting the SPEED field to a value other than 0h (i.e., selecting a specific speed) is not recommended. The drive is designed to dynamically select the optimal speed to achieve maximum systemic performance. This is based on complex criteria including interface bandwidth, host throughput, data compressibility, etc.

- **Bytes 4-5**: Reserved
- **Bytes 6-7**: BLOCK DESCRIPTOR LENGTH: \<Per I_T nexus\> (changeable)

  | Value | Description |
  |-------|-------------|
  | 0000h | No block descriptor follows |
  | 0008h | A single block descriptor follows |

## 6.6.1.2 Block Descriptor for Mode Select (6/10)

The presence of the block descriptor in the Mode Select command depends on the value of the Block Descriptor
Length in the mode parameter header. There is one copy of the block descriptor for each initiator. The layout of the block descriptor is the same for Mode Select (6) and Mode Select (10). Mode parameter header and block descriptor policy (see 4.7.1.1 on page 40) describes this devices non-standard behavior related to fields in the block descriptor.

> Note that mounting a volume that modifies the value of fields in the block descriptor does not establish a unit
> attention condition.

```
Table 325 — Block Descriptor for Mode Select

                                                         Bit
Byte
           7 msb       6         5         4         3         2         1      0 lsb
  0                                        DENSITY CODE

  1        MSB
                                         NUMBER OF BLOCKS
  3                                                                             LSB
  4                                          Reserved
  5        MSB
                                           BLOCK LENGTH
  7                                                                             LSB
```

The block descriptor field definitions follow:

- **Byte 0**: DENSITY CODE: REPORT DENSITY SUPPORT - 44h (see 5.2.26 on page 138) provides additional information (changeable-ignored)

  | Value | Description |
  |-------|-------------|
  | 00h | The density is unidentified (no medium present or unknown/unsupported medium present) |
  | 40h | Ultrium 1 density |
  | 42h | Ultrium 2 density |
  | 44h | Ultrium 3 density |
  | 46h | Ultrium 4 density |
  | 58h | Ultrium 5 density |
  | 5Ah | Ultrium 6 density |
  | 5Ch | Ultrium 7 density |
  | 5Dh | Ultrium M8 density |
  | 5Eh | Ultrium 8 density |
  | 60h | Ultrium 9 density |
  | 7Fh | Do not change density (set only - NOOP) |
  | FFh | Use current medium density (set only) |

- **Bytes 1-3**: NUMBER OF BLOCKS: 000000h (non-changeable)
- **Byte 4**: Reserved
- **Bytes 5-7**: BLOCK LENGTH: 000000h \<Per I_T nexus\> (changeable)
  - Any value of block length can be specified between the minimum and the maximum block lengths, inclusive, specified in the Read Block Limits command.
  - A Block Length value of 000000h indicates that the logical block size to be written to or read from the medium must be explicitly specified by the Transfer Length field in the CDB and the fixed bit must be 0b (see 5.2.15, 5.2.41, and 5.2.42). Additionally the read-type overlength ILI reporting is suppressed as described in 4.12.1.

## 6.6.2 Mode Parameter List for Mode Sense (6/10)

For Mode Sense (6) the mode pages are preceded by a 4-byte mode parameter header below and an optional
block descriptor. If the DBD field is 0b, an 8-byte block descriptor follows the mode parameter header (see
6.6.2.2). If the DBD field is 1b, the block descriptor is not present and the first mode page follows the mode parameter header.

```
Table 326 — Mode Parameter List for Mode Sense (6)

                                                    Bit
Byte
         7 msb     6         5         4         3         2         1    0 lsb
  0
                                      Mode Parameter Header
  3
  4                                    Block Descriptor
4+n-1         (if DBD = 0b, then n = 8 and m=12, else (i.e., DBD = 1b), n = 0, and m = 4)

  m
                                        Mode Pages
  p
```

For Mode Sense (10) the mode pages are preceded by an 8-byte mode parameter header (see 6.6.2.1) and an optional block descriptor. If the DBD field is 0b, an 8-byte block descriptor follows the mode parameter header (see 6.6.1.2). If the DBD field is 1b, the block descriptor is not present and the first mode page follows the mode parameter header. Mode page descriptions begin at MP 01h: Read-Write Error Recovery on page 370.

```
Table 327 — Mode Parameter List for Mode Sense (10)

                                                    Bit
Byte
         7 msb     6         5         4         3         2         1    0 lsb
  0
                                      Mode Parameter Header
  7
  8                                    Block Descriptor
8+n-1         (if DBD = 0b, then n = 8 and m=16, else (i.e., DBD = 1b) n = 0, and m = 8)

  m
                                        Mode Pages
  p
```

There is one copy of the mode parameter header for each initiator.

  1                                   MEDIUM TYPE

```
Table 329 — Mode Parameter Header for Mode Sense (10)

                                                    Bit

Mode parameter header field descriptions follow:

- **Bytes 0-1**: MODE DATA LENGTH - The length in bytes of the following data that is available to be transferred. ## 6.6.3 Mode Page Layout

```
Table 331 — Mode Page Layout

                                                         Bit
Byte
           7 msb       6         5         4         3         2         1      0 lsb

```

```
Table 332 — Mode Page Subpage Layout

                                                         Bit
Byte
           7 msb       6         5         4         3         2         1      0 lsb
  0        PS       SPF(1b)                         PAGE CODE

  1                                      SUBPAGE CODE

  2       (MSB)
                                      PAGE LENGTH (n-3)
  3                                                                            (LSB)
  4
                                       Mode Parameters
  n
```

The individual mode page descriptions that follow include the field descriptions. Each field is non-changeable
unless specifically identified otherwise.

For those mode pages that allow saving of at least one parameter, the PS (Parameter Saveable) bit is returned in
a MODE SENSE with a value of 1b. For those mode pages that do not allow saving of at least one parameter, the
PS bit is returned in a MODE SENSE with a value of 0b. The PS bit is ignored in the parameter data transferred
with a MODE SELECT command.

## 6.6.4 Supported Mode Pages

The following standards-based mode pages are supported:

- MP 01h: Read-Write Error Recovery (see 6.6.5 on page 370)
- MP 02h: Disconnect-Reconnect (see 6.6.6 on page 371)
- MP 0Ah: Control (see 6.6.7 on page 373)
- MP 0Ah[01h]: Control Extension (see 6.6.8 on page 374)
- MP 0Ah[F0h]: Control Data Protection (see 6.6.9 on page 375)
- MP 0Fh: Data Compression (see 6.6.10 on page 377)
- MP 10h: Device Configuration (see 6.6.11 on page 379)
- MP 10h[01h]: Device Configuration Extension (see 6.6.12 on page 381)
- MP 11h: Medium Partition Page (see 6.6.13 on page 384)
- MP 18h: Protocol-Specific Logical Unit (see 6.6.14 on page 389)
- MP 19h: Protocol specific port (see 6.6.15 on page 390)
- MP 1Ah: Power Condition (see 6.6.16 on page 393)
- MP 1Ch: Informational Exceptions Control (see 6.6.17 on page 394)
- MP 1Dh: Medium Configuration (see 6.6.18 on page 397)

The following vendor-specific mode pages are also supported:

- MP 24h: Vendor-Specific (see 6.6.19 on page 398)
- MP 2Fh: Behavior Configuration (see 6.6.20 on page 401)
- MP 30h: Device Attribute Settings (see 6.6.21 on page 404)
- MP 30h[01h]: Drive MAC address - Device attribute settings (see 6.6.21.3.2 on page 409)
- MP 30h[02h]: Drive IP address and subnet mask - Device attribute settings (see 6.6.21.3.3 on page 411)
- MP 30h[20h]: Encryption mode - Device Attribute Settings (see 6.6.21.4.1 on page 413)
- MP 30h[40h]: SkipSync - Device attribute settings (see 6.6.21.5.1 on page 415)
- MP 30h[42h]: End of partition behavior control - Device attribute settings (see 6.6.21.5.2 on page 418)
- MP 30h[43h]: Feature switches - Device attribute settings (see 6.6.21.5.3 on page 419)
- MP 30h[44h]: Preferred Cartridge Type - Device attribute settings (see 6.6.21.5.4 on page 421)
- MP 3Eh: Engineering Support (see 6.6.22 on page 423)
- Mode Page 3Fh: All Pages

> NOTE 46 - Mode page 3Eh is for engineering use only and is not included in page 3Fh
