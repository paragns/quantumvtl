# 6.3.14 IP C0h: Drive Component Revision Levels

- **INQUIRY - 12h** (see 5.2.5 on page 84) describes how to request this page.

## 6.3.14.1 Returned Data - IP C0h: Drive Component Revision Levels

For LUN 0, the following data is returned:

```
Table 220 — IP C0h: Drive Component Revision Levels

                                                Bit
Byte    7 msb       6           5           4           3           2           1       0 lsb
  0     PERIPHERAL QUALIFIER (000b)                     PERIPHERAL DEVICE TYPE (01h)
  1                                 PAGE CODE (C0h)
  2                               Reserved
  3                           PAGE LENGTH (27h)
  4
                                CODE NAME
  15
  16
                            TIME (HHMMSS)
  22
  23
                            DATE (YYYYMMDD)
  30
  31
                                PLATFORM
  42
```

- **Byte 0** - Peripheral Data:
  - Bits 7-5: Peripheral Qualifier: 000b
  - Bits 4-0: Peripheral Device Type: 01h
- **Byte 1** - Page Code: C0h
- **Byte 2** - Reserved
- **Byte 3** - Page Length: 27h
- **Bytes 4-15** - CODE NAME: The code name definition is not published.
- **Bytes 16-22** - TIME: Time the code was built in HHMMSS format with a trailing NULL (i.e., 00h). Prior to May 2010, this field was set to ASCII zeros (i.e., 30h).
- **Bytes 23-30** - DATE: The date the code was built.
- **Bytes 31-42** - PLATFORM: ASCII characters containing the `<protocol>_<package>[_<variant>]` for the drive. This is left-aligned with ASCII spaces padded at the end. This is set to ASCII zeros on Ultrium drive platforms released prior to May 2010.

| Symbol | Value | Description |
|--------|-------|-------------|
| \<protocol\> | sas | Serial Attached SCSI (SAS) |
| \<protocol\> | fcp | Fibre Channel (FC) |
| \<package\> | fh | Full height |
| \<package\> | hh | Half height |
| \<package\> | hl | Half height V2 |
| \<variant\> | f | FIPS |
